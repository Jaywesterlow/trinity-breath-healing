# Requirements: Trinity Breath & Healing

**Defined:** 2026-06-14
**Last revised:** 2026-06-15 (Vercel swap + design-driven overrides from Phase 0 CONTEXT.md)
**Core Value:** SEO + AEO discoverability of trustworthy, citeable content — found and *cited* by Google, AI Overviews, ChatGPT, Perplexity, Claude, Apple Intelligence on Dutch breathwork / trauma-release / energetic-healing queries. Visual fidelity matters, but never at the expense of SEO/AEO.

## v1 Requirements

Categories derived from research + Figma frames + Phase 0 context.

### Foundation (FND)

- [ ] **FND-01**: SvelteKit + Svelte 5 (runes) + TS strict scaffolded with `@sveltejs/adapter-vercel` (region `fra1`); SSG via `export const prerender = true` in root `+layout.ts`
- [ ] **FND-02**: Plain CSS architecture — single `static/global.css` for CSS variables (design tokens), CSS reset, typography baseline, truly-global utility classes; every other style lives inside its respective `.svelte` component file in a `<style>` block (Svelte scopes automatically). NO Tailwind, NO CSS-in-JS, NO SCSS/SASS, NO shared CSS files beyond `global.css`.
- [ ] **FND-03**: `mdsvex` configured for `.svx` files in `src/content/`; Vite glob imports for collection-style access; Zod schemas for type-safe frontmatter
- [ ] **FND-04**: Design tokens placeholder set (CSS variables for color, spacing, radius, type, motion) marked `/* TODO */`; real values from Figma provided in a later phase
- [ ] **FND-05**: Mobile-first reset + responsive grid system
- [ ] **FND-06**: Self-hosted woff2 fonts with `font-display: swap` + metric-matched fallback (no CLS)
- [ ] **FND-07**: `SITE_URL` env strategy with no localhost default (resolves to Vercel-default URL in v1)
- [ ] **FND-08**: Reserved 200-response stub pages exist from day one for all 14 routes (`/werkwijze`, `/over-mij`, `/behandelingen`, `/contact`, `/diensten/{mahatma-healing,goldhealing,raster-energie,spinal-touch}`, `/diensten`, `/blog`, `/artikelen`, `/faq`, `/privacyverklaring`, `/algemene-voorwaarden`) — zero-301 v2 migration guarantee
- [ ] **FND-09**: Repository on GitHub at `trinity-breath-healing` (private), main-branch-protected, Vercel auto-deploys from main, preview URLs auto-generate per PR
- [ ] **FND-10**: GitHub Actions workflow runs build + Lighthouse CI + pa11y + JSON-LD validation on every PR; merge blocked on failure

### SEO Primitives (SEO)

- [ ] **SEO-01**: `+layout.svelte` (root) + `<Head>` Svelte component (wrapping `<svelte:head>`) enforce per-page unique `<title>` (50–60 chars) and meta description (150–160 chars) in initial HTML
- [ ] **SEO-02**: `<PageTitle>` Svelte component enforces exactly one `<H1>` per page; logical H2/H3 hierarchy; CI gate post-build parses rendered HTML to confirm
- [ ] **SEO-03**: Semantic HTML5 landmarks (`<main>`, `<nav>`, `<article>`, `<section>`, `<footer>`) on every page
- [ ] **SEO-04**: Canonical tag on every page
- [ ] **SEO-05**: `hreflang="nl"` + `x-default` infrastructure ready (Dutch ships in v1; EN deferred)
- [ ] **SEO-06**: Open Graph + Twitter Card meta on every page
- [ ] **SEO-07**: XML sitemap generated by `src/routes/sitemap.xml/+server.ts` endpoint (hand-rolled); referenced in robots.txt
- [ ] **SEO-08**: `robots.txt` with explicit `Allow: /` blocks for `OAI-SearchBot`, `ChatGPT-User`, `PerplexityBot`, `Perplexity-User`, `ClaudeBot`, `Claude-User`, `Google-Extended`, `Applebot-Extended` placed BEFORE wildcard `User-agent: *`, plus sitemap reference
- [ ] **SEO-09**: Visible `dateModified` on the landing page (recency-citation signal)
- [ ] **SEO-10**: Search Console URL-prefix verification via HTML-file method against Vercel-default URL (DNS-TXT deferred until custom domain connected)
- [ ] **SEO-11**: CI gate fails the build if any page has zero or multiple H1s, missing or duplicate meta, or invalid canonical

### Structured Data / AEO (SCH)

- [ ] **SCH-01**: `<JsonLd>` Svelte component emits single `<script type="application/ld+json">` per page (via `<svelte:head>`) using `schema-dts` types
- [ ] **SCH-02**: `Organization` + `ProfessionalService` shared schema source-of-truth file (NOT `LocalBusiness` — practice has no customer storefront)
- [ ] **SCH-03**: `Person` schema for the practitioner (E-E-A-T identity)
- [ ] **SCH-04**: Per-modality `Service` schemas (Mahatma Healing, Goldhealing, Raster Energie, Spinal Touch) with `areaServed` (Amsterdam, Zaandam, Almere, Weesp, Oostzaan, Hoofddorp, Badhoevedorp) and `provider` linked to Organization
- [ ] **SCH-05**: `WebSite` schema with site search action (type-ready, search disabled in v1)
- [ ] **SCH-06**: `BreadcrumbList` schema on all non-root pages
- [ ] **SCH-07**: `FAQPage` schema generated from the same source as the rendered inline FAQ section under contact (single source of truth)
- [ ] **SCH-08**: CI gate validates JSON-LD parse + `schema-dts` types + Rich Results Test on every build

## Out of Scope

Explicit exclusions to prevent scope creep.

| Feature | Reason |
|---------|--------|
| Client-side rendering / SPA | Blinds AI crawlers; violates Core Value (PITFALLS #1) |
| `llms.txt` / `llms-full.txt` | No proven retrieval effect per reference doc (2026 verified) |
| GA4 analytics | Requires cookie banner → CLS hit + US data residency |
| Cloudflare Pages hosting | User chose Vercel (2026-06-15) |
| Calendly | User chose Cal.com (research recommendation; open-source + EU posture) |
| Calendly/Cal.com inline at top of page | LCP regression (PITFALLS #8) |
| Separate `/boeken` page | Booking is inline in landing contact-section toggle per Figma Frame 5 |
| Testimonials on landing | User direction 2026-06-15: no reviews in v1 |
| `/reviews` route | User direction 2026-06-15: drop entirely |
| Aggregate review schema | No reviews in v1 |
| Practitioner photograph on landing | Figma uses illustration; no photo blocker |
| Formspree / Tally / Typeform | Black-box form relays, US data residency, opaque GDPR |
| reCAPTCHA | Heavy JS + privacy posture; honeypot + rate-limit sufficient |
| Pop-ups, intrusive cookie banner | UX + CWV hit; cookieless stack |
| Autoplay hero video | LCP killer + a11y regression |
| JS-loaded testimonials | Invisible to AI crawlers (moot — no testimonials anyway) |
| Embedded YouTube as primary content | Heavy iframe |
| Stock wellness photography | N/A — Figma uses illustration |
| User accounts / login | Booking flow is anonymous |
| E-commerce | Practice sells time, not products |
| Native mobile apps | Web-first |
| Newsletter signup (v1) | v2+ |
| Live chat widgets | LCP/INP hit |
| Service workers (v1) | Premature complexity |
| Client-side language switcher | Forces JS; breaks `hreflang` discipline |
| "Genezing" / therapeutic-cure language | Wet BIG legal risk |
| Before/after medical claims | Wet BIG + Reclame Code risk |
| Mock booking timezone | Real `Europe/Amsterdam` only; DST tests required |
| Custom domain registration in v1 | User direction: not using domain yet; ship on `*.vercel.app` |
| `LocalBusiness` schema with public storefront address | Practice is mobile + remote + part-time; no customer storefront |
| Tailwind CSS (any version) | User direction 2026-06-15: plain CSS only |
| CSS-in-JS, SCSS/SASS, PostCSS preprocessing beyond Vite defaults | User direction 2026-06-15: plain CSS only |
| Shared CSS files beyond `static/global.css` | User direction 2026-06-15: per-component CSS lives inside the component `<style>` block |
| `src/styles/` directory | User direction 2026-06-15: single `static/global.css` + per-component styles |
