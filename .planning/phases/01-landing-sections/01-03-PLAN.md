Script: `import { onMount } from 'svelte'; let sectionEl: HTMLElement;`

onMount (D-13 sibling-close pattern):
```typescript
onMount(() => {
  const allDetails = Array.from(sectionEl.querySelectorAll('details'));
  allDetails.forEach((detail) => {
    detail.addEventListener('toggle', () => {
      if (detail.open) {
        allDetails.forEach((other) => {
          if (other !== detail && other.open) other.open = false;
        });
      }
    });
  });
});
```

IMPORTANT desktop accordion: On desktop, the <details> elements should behave as plain cards. Approach: in the onMount function, if window.matchMedia('(min-width: 1024px)').matches, set all details.open = true. Also add a resize listener to re-open all details when viewport crosses 1024px. The CSS ensures they look like cards on desktop (no chevron, pointer-events:none on summary).

Alternatively (simpler): use CSS to unconditionally display the .card-body on desktop:
```css
@media (min-width: 1024px) {
  details.card .card-body { display: block !important; }
  details.card summary { pointer-events: none; }
  .card-heading .chevron { display: none; }
}
```
The `display: block !important` on .card-body overrides the UA stylesheet that hides details content when closed. This is the correct CSS approach — no JS needed for desktop open state.
