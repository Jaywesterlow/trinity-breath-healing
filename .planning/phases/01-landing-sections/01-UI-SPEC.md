## Spacing Scale

Tokens declared in `static/global.css` — use exclusively; never hardcode px values in component styles.

| Token | Value | Usage |
|-------|-------|-------|
| `--space-1` | 4px (0.25rem) | Icon gaps, tight inline padding |
| `--space-2` | 8px (0.5rem) | Compact element spacing, icon-label gap |
| `--space-3` | 12px (0.75rem) | Dense list item spacing |
| `--space-4` | 16px (1rem) | Default element spacing, form field gap |
| `--space-5` | 20px (1.25rem) | Card inner padding |
| `--space-6` | 24px (1.5rem) | Container inline padding, section inner spacing |
| `--space-8` | 32px (2rem) | Card gap, grid gap |
| `--space-10` | 40px (2.5rem) | Sub-section separation |
| `--space-12` | 48px (3rem) | Section top/bottom padding (mobile) |
| `--space-16` | 64px (4rem) | Section top/bottom padding (desktop) |

**Exceptions:**
- Touch targets: minimum 44×44px on all interactive elements (A11Y-03). Use padding to reach minimum — do not scale font or change layout.
- Header height: derived from content + `--space-4` top/bottom; `padding-top` on `<main>` set to match at runtime (D-06).
- Container: `max-width: var(--container-max)` (1200px) + `padding-inline: var(--space-6)` (D-37).

## Typography

All type sizes use tokens from `static/global.css`. Never hardcode sizes in components.

### Roles and Tokens

| Role | Token | Computed Value | Font | Weight | Line Height |
|------|-------|----------------|------|--------|-------------|
| Body / base / card body | `--font-size-base` | 16px (1rem) | DM Sans | 400 (`--font-weight-regular`) | 1.5 (`--line-height-normal`) |
| Small / label / eyebrow / caption | `--font-size-sm` | 14px (0.875rem) | DM Sans | 400 (body) / 700 (eyebrow) | 1.5 |
| Section heading (H2) | `--font-size-2xl` | clamp(1.5rem, 4vw, 2rem) | Cormorant Garamond | 700 | 1.2 (`--line-height-tight`) |
| Page H1 / display | `--font-size-3xl` | clamp(2rem, 6vw, 3rem) | Cormorant Garamond | 700 | 1.2 |

**Rules:**
- H1–H6 always use `--font-display` (Cormorant Garamond), `--line-height-tight`, `--font-weight-bold`.
- Body prose always uses `--font-body` (DM Sans), `--line-height-normal`.
- Exactly one `<h1>` per page (SEO-11). Landing H1: "Rust in je hoofd. Ontspanning in je lichaam." (LND-01).
- Eyebrow labels (e.g. "Werkwijze", "Over mij", "Behandelingen", "Contact", "FAQ") use `--font-size-sm`, `--font-weight-bold`, `--color-muted`, `letter-spacing: 0.08em`, `text-transform: uppercase`. The uppercase + letter-spacing modifiers visually differentiate eyebrow from label at the same token size.
- Card headings (h3 / `<summary>`) use `--font-size-base`, `--font-weight-bold`, DM Sans — bold weight provides hierarchy without a separate size role.

## Color

All tokens from `static/global.css`. Never use hex literals in component styles.

| Role | Token | Value | Usage |
|------|-------|-------|-------|
| Dominant surface (60%) | `--color-bg-sand` | `#FAF0E6` | Page background, hero section, FAQ section background |
| Secondary surface (30%) | `--color-card-warm` | `#EDE8D0` | Card backgrounds (Werkwijze cards, Behandelingen cards, footer background) |
| Primary text / brand dark | `--color-fg-forest` | `#3D4A35` | All body text, headings, nav links, footer text |
| Accent (10%) | `--color-accent-gold` | `#D4A968` | **Reserved for:** CTA buttons ("Maak een afspraak", "Verstuur email"), focus rings (2px outline), active carousel card indicator, radio toggle active state, stat number highlights |
| Secondary text / borders | `--color-muted` | `#7A8C6E` | Eyebrow text, secondary body copy, inactive nav items, placeholder text, carousel inactive card text |
| Input / divider borders | `--color-border` | `#B68A6B` | Form field borders, section dividers, card borders |
| Destructive | none in Phase 1 | — | No destructive actions in Phase 1 (form submit errors use `--color-border` + inline text, not red) |

**Accent reserved-for list (explicit):** CTA button fills, focus-visible outline rings, active carousel card accent border or indicator dot, active radio toggle pill, stat number text color. Nothing else uses `--color-accent-gold`.

**Header background:** `--color-bg-sand` (solid, always — no transparent-over-hero per D-05).

**Mobile overlay background:** `--color-fg-forest` (full-screen green overlay per D-08 design intent; white text on overlay).

### LND-02 — Header + Mobile Menu

**Header behavior:**
- `position: fixed; top: 0; left: 0; right: 0; z-index: 100`.
- Smart-hide: hides (translateY -100%) on scroll-down > 8px threshold; re-appears on scroll-up. Svelte `onMount` scroll listener (D-05).
- Background: `--color-bg-sand`. No transparency. `box-shadow` on scroll (thin, `--color-border` tint).
- `<main>` gets `padding-top` equal to header height (D-06). Header height: auto (content-derived); use `clientHeight` from scroll listener.

**Logo:** `<img src="/logo.svg">` (SVG, not `EnhancedImage` — SVGs are not raster). Alt: "TRINITY Breath & Healing logo". Source: `src/lib/assets/logo.svg` (user-provided, D-09).

**Desktop nav (≥ 1024px):**
- Nav links: Home · Werkwijze · Over mij · Behandelingen · Contact. `--font-size-sm`, `--color-fg-forest`, `--font-weight-regular`.
- Active/hover: `--color-accent-gold` underline (2px, `text-decoration`).
- CTA "Maak een afspraak": same button style as hero CTA (smaller padding variant: `--space-2` block, `--space-5` inline).
- Social icons: Lucide/Heroicons SVG, size 20×20px. `--color-fg-forest`. Hover: `--color-accent-gold`. Each link MUST have `aria-label`: Instagram → `"Volg ons op Instagram"`, Facebook → `"Volg ons op Facebook"`, X → `"Volg ons op X"`.

**Mobile hamburger (< 1024px):**
- Hamburger button: 44×44px touch target, `--color-fg-forest` icon.
- Opens full-screen overlay (D-08). Overlay: `background: var(--color-fg-forest)`. Text: `--color-bg-sand` (white-ish).
- Overlay closes via: X button (44×44px, top-right) OR Escape key (D-08 — WCAG 2.2 AA A11Y-01).
- Overlay has `role="dialog" aria-modal="true" aria-label="Navigatiemenu"`.
- Nav links in overlay: `--font-size-2xl`, Cormorant Garamond, centered. CTA button at bottom.
- Overlay transition: fade + translateY (respects `prefers-reduced-motion`).

**Motion:** hide/show header uses `transition: transform var(--motion-base) var(--ease-out)`. Overlay fade: `var(--motion-slow) var(--ease-out)`.

## Interaction Contracts

### Scroll Behaviors
- Header smart-hide: hide on scroll-down > 8px delta, show on scroll-up. Transition: `transform var(--motion-base) var(--ease-out)`.
- Smooth scroll to anchor IDs (`#werkwijze`, `#over-mij`, `#behandelingen`, `#contact`) via `scroll-behavior: smooth` on `<html>` — disabled when `prefers-reduced-motion: reduce` (global.css rule already present).

### Focus Management
- Mobile overlay open: focus moves to X close button (first focusable element in overlay).
- Mobile overlay close: focus returns to hamburger button.
- `<details>` accordion: native browser focus behavior (no override needed).

### Reduced Motion
All transitions and animations MUST be wrapped in `@media (prefers-reduced-motion: reduce)` suppression (already in `static/global.css` via `animation-duration: 0.01ms` override). Carousel JS must also skip animation when `window.matchMedia('(prefers-reduced-motion: reduce)').matches`.

### WCAG 2.2 AA Touch Targets
- All interactive elements: minimum 44×44px (A11Y-03). Enforce with `min-height: 44px; min-width: 44px` or sufficient padding.
- Focus-visible: `outline: 2px solid var(--color-accent-gold); outline-offset: 2px` (already in global.css reset).
