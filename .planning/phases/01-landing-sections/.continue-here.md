# BLOCKING CONSTRAINTS — Read Before Anything Else

> These are not suggestions. Each constraint below was discovered through failure.
> Acknowledge each one explicitly before proceeding.

- [ ] CONSTRAINT: CMD-ampersand — npm/npx/prettier.cmd scripts fail when invoked from the project directory because `Trinity Breath & Healing` contains `&` which CMD interprets as a command separator. Use `node node_modules/prettier/bin/prettier.cjs` or PowerShell `& node $prettier ...` directly.
- [ ] CONSTRAINT: Executor agents load permissions at startup — permissions added to `settings.local.json` mid-session are NOT seen by already-running task agents. Always start fresh session after editing permissions.

**Do not proceed until all boxes are checked.**

## Critical Anti-Patterns

| Pattern | Description | Severity | Prevention Mechanism |
|---------|-------------|----------|---------------------|
| CMD ampersand path split | `&` in project folder name breaks all `.cmd` batch wrappers (npm, npx, prettier.cmd, etc.) — path gets truncated at `&` | blocking | Always invoke via PowerShell with `node path\to\bin\script.cjs` directly; never use `.cmd` wrappers for this project |
| Stale executor permissions | Task agents spawned before permissions edit ignore the updated `settings.local.json` | advisory | Edit permissions file FIRST in a session, then spawn executors; never edit mid-flight |
| Importing deleted components | Building without checking git status causes silent build failures when pages import deleted component files | advisory | Always run `git status` before starting work on components |
